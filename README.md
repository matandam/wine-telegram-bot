# Wine Education Bot

A production-ready Telegram bot for wine education. Users discover it on Telegram, send `/start`, and get on-demand wine lessons covering 60 regions across 15 countries — plus daily morning deliveries, deep dives on any region or grape, and personalised recommendations.

---

## What This Bot Does

- **On-demand lessons** via `/lesson` — each delivery covers a new wine region, cycling through all 60 until the user completes the full curriculum
- **Region deep dives** via `/region [name]` — detailed lessons on any named wine region
- **Grape deep dives** via `/grape [name]` — varietal education on any named grape
- **Daily subscription** via `/daily` — opt-in morning lessons at 9:00 AM in the user's chosen timezone
- **Personalised recommendations** via `/recommend` — 3-question flow (colour, style, occasion) producing tailored bottle suggestions
- **Progress tracking** via `/mystats` — lessons received, regions covered by country, subscription status

## What This Bot Does NOT Do

- **No general AI conversation.** Any non-command text message receives: *"I am a wine education bot. Use /help to see what I can do."*
- **No open-ended chat.** There are no free-form Q&A capabilities.
- **No personal data beyond what you send.** Stored: Telegram ID, username, first name, timezone, lesson history. No message content is stored.
- **No shell access, web browsing, or file system access** beyond its own `data/` directory.
- **No financial transactions** of any kind.

## Setup

### Prerequisites

- Node.js 18+
- A Telegram bot token from [@BotFather](https://t.me/botfather)
- An Anthropic API key from [console.anthropic.com](https://console.anthropic.com)

### Install

```bash
git clone <repo>
cd wine-telegram-bot
npm install
```

### Configure

```bash
cp .env.example .env
```

Edit `.env`:

```
TELEGRAM_BOT_TOKEN=your_bot_token_from_botfather
ANTHROPIC_API_KEY=your_anthropic_api_key
```

Optional overrides:
```
ANTHROPIC_MODEL=claude-opus-4-6   # default
DB_PATH=./data/wine_bot.db        # default
```

### Run

**Development (ts-node):**
```bash
npm run dev
```

**Production:**
```bash
npm run build
npm start
```

## Architecture

```
src/
  index.ts          — Bot setup, polling, command routing, message handler
  db.ts             — SQLite schema, prepared statements, CRUD operations
  regions.ts        — Region list loader, per-user next-region selection
  lessons.ts        — Anthropic API calls for lesson/grape/recommendation content
  scheduler.ts      — node-cron job for daily 9 AM delivery
  commands/
    start.ts        — /start: welcome + user registration
    help.ts         — /help: command reference
    daily.ts        — /daily: subscription + timezone selection flow
    stop.ts         — /stop: unsubscribe
    lesson.ts       — /lesson: deliver next unseen region lesson
    region.ts       — /region [name]: region deep dive
    grape.ts        — /grape [name]: grape deep dive
    recommend.ts    — /recommend: 3-step preference → personalised suggestion
    mystats.ts      — /mystats: progress display
data/
  regions.json      — 60 region definitions (name, country, index)
  wine_bot.db       — SQLite database (auto-created on first run)
```

### Database Schema

```sql
-- Registered users
users (id, telegram_id, username, first_name, subscribed, timezone, created_at)

-- Per-user delivery history (prevents duplicate lessons)
lesson_history (id, telegram_id, region_index, delivered_at)
  UNIQUE(telegram_id, region_index)

-- Multi-step flow state (daily timezone, recommend questions)
user_state (telegram_id, state JSON)
```

### AI Content Generation

Lessons are generated by the Anthropic Claude API (default: `claude-opus-4-6`) with a tightly scoped system prompt that restricts output to wine education content only. Adaptive thinking is enabled for nuanced region analysis.

Each full lesson covers:
1. Region overview (terroir, climate, soils, geography)
2. Wine style (grapes, body, acidity, tannin, aging)
3. History & cultural context
4. Sensory breakdown (aromas, flavors, texture, finish)
5. 3–5 bottle recommendations with price tier
6. Food pairing & serving guidance
7. Atmospheric immersive taste narrative (150–250 words)
8. Key takeaways

## Safety Notes

- The bot only responds to defined commands and registered multi-step flows
- Any free-text message outside a flow is rejected with a scope message
- The Claude system prompt is scoped to wine content only; it explicitly blocks off-topic content
- No user message content is stored in the database
- The bot does not follow links or execute user-provided strings as commands
- Rate limiting: the scheduler delivers at most one lesson per user per 20 hours

## Deployment

The bot uses long polling (no webhook required) and is suitable for running on any VPS, a Raspberry Pi, or a cloud instance. The SQLite database is a single file in `data/` — back it up regularly.

For a lightweight production deployment:
```bash
npm run build
node dist/index.js
```

Use a process manager like `pm2` for auto-restart:
```bash
npm install -g pm2
pm2 start dist/index.js --name wine-bot
pm2 save
```
